# خانه ایده‌آل — وب‌سایت مشاور املاک

یک صفحه فرود (Landing Page) کامل، ریسپانسیو و آماده استقرار برای یک مشاور املاک؛ ساخته‌شده فقط با **HTML / CSS / JavaScript خالص** (بدون فریم‌ورک، بدون نیاز به فرآیند build).

## ساختار پروژه

```
idealhome/
├── index.html              صفحه اصلی
├── 404.html                صفحه خطای ۴۰۴ (اختیاری، برای هاست‌های استاتیک)
├── robots.txt               راهنمای خزنده‌های موتور جست‌وجو
├── sitemap.xml               نقشه سایت برای سئو
├── site.webmanifest          متادیتای PWA (آیکون، رنگ تم و...)
├── css/
│   └── style.css             تمام استایل‌ها (بدون وابستگی خارجی)
├── js/
│   └── script.js             منوی موبایل، کروسل، آکاردئون، اسلایدر نظرات و ...
└── assets/
    ├── fonts/
    │   └── vazirmatn-variable.woff2   فونت فارسی، به‌صورت لوکال روی سرور خودتان (بدون وابستگی به گوگل فونت)
    └── img/
        ├── logo-mark.svg
        ├── favicon.svg
        ├── hero-villa.svg
        └── property-1.svg … property-6.svg
```

## چرا فونت و آیکون‌ها لوکال هستند؟

برای یک سایت آماده‌ی هاست، وابستگی به سرورهای شخص ثالث (مثل Google Fonts یا CDNهای عکس) یک نقطه‌ضعف واقعی است: ممکن است در برخی شبکه‌ها کند یا فیلتر باشد، یا لینک عکس بعداً از کار بیفتد. به همین دلیل:

- فونت **Vazirmatn** به‌صورت فایل `woff2` در پروژه قرار گرفته و مستقیماً از هاست خودتان سرو می‌شود (لایسنس OFL — رایگان برای استفاده تجاری، فایل لایسنس هم کنار فونت قرار دارد).
- تمام تصاویر (نمای ویلا، آپارتمان‌ها، لوگو) به‌صورت SVG برداری طراحی شده‌اند: وزن فایل بسیار کم، کیفیت کامل روی هر صفحه‌نمایشی (از جمله Retina)، و **هرگز شکسته نمی‌شوند** چون هیچ درخواستی به سرور بیرونی نمی‌زنند.

### جایگزینی با عکس‌های واقعی ملک‌ها

وقتی عکس واقعی از ملک‌های خودتان دارید، کافی است در پوشه‌ی `assets/img/` فایل‌های `property-1.jpg` تا `property-6.jpg` (یا هر نام دیگری) را قرار دهید و در `index.html` مقدار `src` مربوط به هر `<img>` را به مسیر جدید تغییر دهید. مثال:

```html
<!-- قبل -->
<img src="assets/img/property-1.svg" alt="...">

<!-- بعد -->
<img src="assets/img/property-1.jpg" alt="...">
```

توصیه: عکس‌ها را با نسبت ۴:۳ و حداکثر عرض ۸۰۰ پیکسل و فرمت `webp` یا `jpg` فشرده (زیر ۲۰۰ کیلوبایت) آماده کنید تا سرعت لود صفحه حفظ شود.

## استقرار روی هاست (Deploy)

این پروژه کاملاً استاتیک است؛ یعنی هیچ سرور بک‌اند یا دیتابیسی نیاز ندارد. چند گزینه:

### ۱. هاست اشتراکی / cPanel
تمام فایل‌ها (به همراه پوشه‌ها) را داخل `public_html` آپلود کنید (مثلاً با FTP یا File Manager). همین کافی است.

### ۲. Netlify / Vercel / Cloudflare Pages
پوشه پروژه را در یک ریپوی Git قرار داده و به یکی از این سرویس‌ها متصل کنید؛ چون فایل build ندارد، فقط کافی است Output/Publish directory را روی ریشه‌ی پروژه (`/`) تنظیم کنید.

### ۳. GitHub Pages
فایل‌ها را در شاخه `main` قرار دهید و در تنظیمات ریپو، Pages را از همان شاخه فعال کنید.

## قبل از انتشار نهایی این‌ها را اصلاح کنید

- [ ] در `index.html`، مقادیر `og:url`، `canonical` و در `robots.txt` / `sitemap.xml` دامنه‌ی واقعی خودتان را جای `idealhome.ir` بگذارید.
- [ ] شماره تلفن‌ها، ایمیل و آدرس در بخش «تماس با ما» و فوتر را به‌روزرسانی کنید.
- [ ] لینک نقشه در فوتر (`https://maps.google.com/?q=...`) را با مختصات واقعی دفتر خود جایگزین کنید.
- [ ] عکس‌های SVG را در صورت تمایل با عکس‌های واقعی ملک جایگزین کنید (بالا توضیح داده شد).
- [ ] فرم جستجو در حال حاضر فقط به بخش املاک اسکرول می‌کند؛ برای اتصال به یک بک‌اند واقعی (فیلتر کردن ملک‌ها)، `id="searchForm"` در `js/script.js` را به API خودتان وصل کنید.

## ویژگی‌های فنی

- ریسپانسیو کامل از موبایل کوچک (۳۲۰px) تا دسکتاپ بزرگ
- بدون وابستگی به هیچ فریم‌ورک یا کتابخانه خارجی
- آکاردئون سوالات متداول با تگ بومی `<details>/<summary>` — حتی بدون اجرای جاوااسکریپت هم کار می‌کند
- منوی موبایل، کروسل املاک، اسلایدر نظرات با جاوااسکریپت خالص (Vanilla JS)
- رعایت دسترس‌پذیری (Accessibility): لینک Skip to content، برچسب‌های aria، فوکوس قابل مشاهده با کیبورد، احترام به `prefers-reduced-motion`
- متادیتای سئو کامل: Open Graph، Twitter Card، JSON-LD (Schema.org RealEstateAgent)، `robots.txt`، `sitemap.xml`
- استایل چاپ (Print CSS) برای چاپ تمیز صفحه در صورت نیاز
